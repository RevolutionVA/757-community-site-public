**Connecting Claude to Everything: The MCP Deep Dive**
Claude is smart. But right now, it's smart in a room with no doors.
It can't check your Slack. It can't read your database. It can't update your Notion. It can't file your GitHub issues. It can't query your Stripe data. Every time you want Claude to work with your actual tools, you're copying and pasting between windows like it's 2019.
MCP (Model Context Protocol) is the fix. It's the universal adapter that connects Claude to Gmail, Slack, Google Drive, Notion, Asana, GitHub, Stripe, databases, and 3,000+ other services — through a single standard protocol. Think of it like USB-C for AI.
This session shows you how to connect Claude to the tools you actually use, with live demos at two levels: Cowork Connectors for everyone (click-to-connect, no code) and Claude Code MCP Servers for developers (install any server from the ecosystem, connect to databases and APIs).
**What we'll cover:**
🔌 **What Is MCP and Why It Matters** The universal adapter that lets Claude talk to any external service. One protocol, thousands of connections.
⚡ **10 Connections We'll Build Live:**
For Everyone (Cowork Connectors — no code):

1. Gmail + Calendar — cross-referencing email and meetings in a single prompt
2. Slack — summarize channels, find un-responded messages, draft replies
3. Google Drive — search, read, and create files across your Drive
4. Notion — query databases, create pages, search your workspace
5. Asana — check overdue tasks, create tasks, update status

For Developers (Claude Code MCP Servers):
6\. GitHub — review PRs\, create issues\, check CI status
7\. Stripe — query customers\, invoices\, MRR trends
8\. Databases — connect to Postgres/Supabase\, run queries in English
9\. Custom APIs — connect Claude to any REST API \(live demo: URL shortener\)
10\. Stacking Connections — one prompt that touches 4\+ services simultaneously

🔍 **Finding and Installing MCP Servers** Where to discover the 3,000+ MCP servers available, how to install them, and how to evaluate which ones are worth adding.

**Who this is for:** Two tracks in one event. If you've attended any of our previous events and want to connect Claude to the services you use every day — this is for you. Developers will learn how to install MCP servers and connect Claude to databases and APIs. Non-technical attendees will learn how to use Cowork Connectors to connect Claude to Gmail, Slack, Drive, Notion, and Asana with zero code.

**Who this is NOT for:** If you haven't installed Claude Desktop yet, start with our free setup guide first. We're spending our time connecting services, not installing.

**What you'll leave with:** ✅ At least 2-3 new services connected to Claude ✅ A free homework doc with all 10 connection prompts and an MCP installation guide ✅ The framework for finding and evaluating MCP servers ✅ An understanding of how to stack multiple connections in a single prompt

**Prerequisites:** — Claude Desktop installed (claude.com/download) — Claude Pro ($20/month) or Max ($100+/month) subscription — Cowork mode accessible — Have login credentials ready for services you want to connect (Gmail, Slack, Notion, Asana, etc.) — Developers: Claude Code CLI installed (npm install -g @anthropic-ai/claude-code)

Author of AI for Business. Currently building at the intersection of AI tooling, agentic workflows, and practical business automation.

Operational Technology defenders are being asked to prioritize risk in environments where accuracy, stability, and time no longer exist in abundance. OT inventories are often incomplete, outdated, or context‑poor, yet they form the foundation for risk mitigation management decisions that carry real safety, uptime, and financial consequences.

At the same time, vulnerability disclosure velocity is accelerating, attacker capabilities are being augmented by AI, and exploitation pathways are adapting faster than traditional defensive models can track. This creates a prioritization problem that is not purely technical, but structural forcing defenders to make high‑stakes decisions with partial visibility. In an AI‑driven world of constant change, static asset lists, scoring models, and patch‑centric workflows struggle to reflect how adversaries actually operate.

This session explores how OT security teams can navigate these contradictions, where prioritization breaks down under pressure, and what must evolve, across inventories, vulnerability management, and organizational decision‑making, to remain effective in the face of continuously shifting threats by reconnecting on the key priorities: why we are here.
