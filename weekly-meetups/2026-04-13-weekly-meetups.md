**Quantum Computing with Q# and Azure Quantum**

Quantum Computing is a rising field in Computer Science that uses the effects of Quantum Mechanics to achieve computations that are very difficult to do or simply impossible to do with regular computer technology. It can be used to simulate complex molecules for research, find optimal solutions for complex datasets, or could eventually break RSA encryption with sufficiently powerful quantum computers. To enter the Quantum Computing field, Microsoft created Q#, a domain specific language for performing quantum computations and Azure Quantum, an Azure service that provides cloud access to quantum simulators and selected real quantum computers.

#### John Morehouse: Exploring Optimized Locking in SQL Server 2025

- **Time:** 06:00 PM
- **Group:** Hampton Roads SQL Server User Group
- **Link:** [Event Link](https://www.meetup.com/hampton-roads-sql-server-user-group/events/314074633/)

**Description:**
**IN PERSON MEETING!!!!!**
Exploring Optimized Locking in SQL Server 2025

When updating data in SQL Server, operators run the gambit of incurring issues if not done correctly. Sometimes, they manifested as lock escalation, the dreaded blocking, disastrous deadlocks, or poor performance. With the release of Optimized Locking, first seen in Azure SQL Database and now in SQL Server 2025, issues are mitigated while helping to enhance performance and transactional concurrency.

In this session, we'll embark on a journey into this new capability, designed to revolutionize concurrency control and improve transaction throughput. We'll examine the underlying mechanism while providing insights into its architecture, implementation, and best practices for maximizing performance.

You will -
• Learn how optimized locking works
• Discover what it requires to function
• Some Gotchas to be aware of

Whether you're a seasoned database professional or a curious developer, this session offers a more profound understanding of SQL Servers's optimized locking feature.

### Saturday, April 18

#### Claude Code Agent Teams Workshop | AICHR

- **Time:** 01:00 PM
- **Group:** 757 Artificial Intelligence Collective
- **Link:** [Event Link](https://www.meetup.com/aicollectivehr/events/314079102/)

**Description:**
***This event is capped at 500 attendees. Register now to secure your spot.***

You've set up Claude Code. You've built your first automations. Now it's time to scale.
Agent Teams is Claude Code's most powerful feature — multiple AI instances working in parallel, sharing task lists, and coordinating directly with each other. Instead of one agent handling everything sequentially, you spin up a team: frontend, backend, QA — each working independently while a team lead orchestrates the whole operation.
This session is a live, demo-driven deep dive into Agent Teams. We're not talking theory. We're spinning up a real multi-agent team on stage and building a project in parallel — live.
If you attended our April 9 event (or watched the recording), this is the next step. If you didn't, you can still follow along — but having Claude Code installed will help.
Join Andrew Stafford for a hands-on look at how to go from "one AI assistant" to "an AI workforce."

🔍 **What We'll Cover**

**Agent Teams Setup** The one-line configuration that unlocks multi-agent orchestration. tmux for split-pane visibility. Display modes. Permission management.

**Live Demo: Multi-Agent Build** We'll spin up a 3-agent team — two builders and a reviewer — and build a real feature in parallel. You'll watch the agents claim tasks, communicate with each other, and coordinate their work in real time.

**Prompt Engineering for Teams** How to define roles, assign file ownership, set communication protocols, and prevent the #1 failure mode (agents overwriting each other's work).

**Plan-Then-Execute at Scale** The pattern that makes Agent Teams reliable: every agent drafts a plan, gets approval, then executes. No wasted tokens, no wrong-direction builds.

**When to Use Teams vs Sub-Agents vs Solo** Agent Teams aren't always the answer. We'll cover the decision framework for choosing the right approach based on your task complexity and budget.

**Token Economics** Agent Teams use 3-5x the tokens of a single session. We'll break down the real costs and show you how to optimize for value.

The gap between "AI assistant" and "AI agent" is closing fast. Claude Code doesn't just suggest code — it plans, executes, verifies, and coordinates. Cowork extends that to knowledge work. Dispatch makes it mobile. Tasks make it persistent.
