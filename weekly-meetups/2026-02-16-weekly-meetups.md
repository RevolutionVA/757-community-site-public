#### Kalyan Inturi-Designing a Central Identity DB Unifying Authentication

- **Time:** 06:00 PM
- **Group:** Hampton Roads SQL Server User Group
- **Link:** [Event Link](https://www.meetup.com/hampton-roads-sql-server-user-group/events/312879970/)

**Description:**
**IN PERSON MEETING!!!!!**
As SaaS platforms grow through acquisitions, identity data often becomes fragmented across multiple databases and user stores, leading to inconsistent access controls, operational complexity, and security risk. This session presents a real-world case study of consolidating authentication and authorization into a centralized, SQL Server‚Äìbacked identity platform used across multiple products.

The solution combined OpenID Connect and OAuth 2.0 with a central identity database, token-time claim enrichment, and event-driven replication to maintain data consistency while supporting near-zero-downtime migrations. The talk focuses on how SQL Server was used to model users, roles, and permissions; support reliable change propagation; and enable auditing, deprovisioning, and session management at scale.

Attendees will learn practical lessons around schema design, transactional consistency, asynchronous replication, and operational monitoring for identity data. The techniques discussed apply broadly to any system where correctness, availability, and security depend on a well-designed and highly reliable SQL Server data layer.
